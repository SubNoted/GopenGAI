### `internal/history`
Conversation history management — tree structure, message editing, context building.

---

### `branch.go`
Handles branch selection, message editing (forking), and session forking.

#### Algorithm
**SelectLeaf**: Verifies leaf exists, belongs to session, is actually a leaf (no children), then updates active_leaf_id in session.

**EditMessage**: Creates new message as sibling of target (same parent), forming new branch. Uses SQL transaction for atomicity. Validates params (role, content length). Updates session active_leaf_id to new message.

**ForkSession**: Copies messages from original session's active branch up to FromMessageID, appends new user message as child of last copied message. Creates new session with parent_session_id pointing to original. Uses SQL transaction.

#### Data
- **Structures**: `EditMessageParams` { SessionID, TargetID, Content, Role }, `ForkSessionParams` { OriginalSessionID, AgentName, Title, FromMessageID, NewContent }
- **Types**: `validRoles` map[string]bool, `maxContentLength` const (100KB)
- **Flow**: SelectLeaf: load leaf → verify session → verify leaf → update active. EditMessage: validate → begin tx → get target → create sibling message → update session active leaf → commit. ForkSession: load original → verify from_message → copy branch → create new session → add new user msg → set active leaf → commit

### `context.go`
Builds context for LLM from active branch.

#### Algorithm
**BuildContext**: Loads active branch via `GetActiveBranch`, converts to `llm.Message`, prepends system prompt, truncates if token count exceeds maxTokens using approximate char-based token estimation (4 chars/token).

**truncateContext**: Drops oldest non-system messages when estimated tokens > maxTokens. Preserves system prompt always.

**estimateTokens**: Rough heuristic — counts message overhead (4 tokens) + content chars/4 + tool call overhead (8 tokens per call).

#### Data
- **Structures**: None exported
- **Types**: `llm.Message` from internal/llm
- **Flow**: Load active branch → convert to agent messages → prepend system prompt → estimate tokens → truncate if needed → return []llm.Message

### `id.go`
Generates unique IDs for messages and sessions.

#### Algorithm
**newID**: Generates random 16-byte hex string formatted as UUID-ish (xxxx-xx-xx-xx-xxxx). Retries crypto/rand 3 times. Falls back to counter + timestamp if entropy depleted (format: fallback-{millis}-{nanosec}-{counter}).

#### Data
- **Structures**: None exported
- **Types**: `idCounter` atomic.Int64
- **Flow**: Try rand.Read 3x → if success return hex string → else return fallback ID with timestamp + monotonic counter

### `repo.go`
Wraps database layer with history-tree-specific operations.

#### Algorithm
**GetActiveBranch**: If session has active_leaf_id, uses recursive CTE query `GetBranchFromRootTo`. Otherwise loads all messages, builds tree, picks longest leaf as default branch.

**GetActiveBranchByLeafID**: Uses recursive CTE query to get root-to-leaf path.

**UpdateActiveLeaf**: Updates session's active_leaf_id using targeted UPDATE (no read-modify-write). Does NOT validate leaf existence.

**branchRowsToMessages**: Converts `db.GetBranchFromRootToRow` to `db.Message`.

#### Data
- **Structures**: `Repository` { q db.Querier, sqldb *sql.DB }
- **Types**: `db.Message`, `db.Session`, `db.CreateMessageParams`, `db.GetAllLeavesParams`
- **Flow**: GetActiveBranch → get session → if active_leaf_id valid → GetBranchFromRootTo else list all msgs → BuildTree → GetLongestLeaf → GetPathFromRoot → return messages

### `tree.go`
Defines `TreeNode` and functions to build, navigate, mutate conversation tree.

#### Algorithm
**BuildTree**: Indexes messages by ID, links children to parents via parent_id, collects roots (messages with no parent or orphan parent), assigns depths via BFS.

**FindNode**: BFS search across all roots for node by message ID.

**GetPathFromRoot**: Walks up from node to root using Parent pointers, reverses order (root first, leaf last).

**GetLongestLeaf**: Returns leaf with deepest depth (longest path).

**InsertNode**: Adds new message as child of parent node in tree (mutates in-place).

**ToAgentMessages**: Converts db.Message to agent.Message, mapping fields by role (user, assistant, tool, system).

#### Data
- **Structures**: `TreeNode` { Message db.Message, Parent *TreeNode, Children []*TreeNode, Depth int }
- **Types**: `db.Message`
- **Flow**: BuildTree: index msgs → link parents/children → collect roots → assign depths → return roots. GetPathFromRoot: walk up Parent → reverse → return path. ToAgentMessages: iterate messages → switch on role → map fields → return []agent.Message
