### `agent/`

#### Algorithm
Loader parses `.md` files with YAML frontmatter â†’ builds `Agent` struct (name, system prompt, tools, permissions). Registry stores agents in-memory, concurrency-safe. ### `api/`
HTTP handlers, routing, session CRUD, chat, OpenAI compatibility.

#### Algorithm
Session CRUD: create (UUID fallback), list, get (+messages), delete (cascade). Chat flow: transaction (read, save user, update status) â†’ LLM call â†’ transaction (save assistant, update) â†’ return response. Error handling: logs + 500/502 responses. ### `history/`
Conversation history tree, message editing, context building.

#### Algorithm
**SelectLeaf**: Verify leaf, update active_leaf_id. **EditMessage**: Create sibling message (new branch), transaction, update active leaf. **ForkSession**: Copy branch up to `FromMessageID`, append new user message, create new session. **BuildContext**: Load active branch â†’ convert to `llm.Message` â†’ prepend system prompt â†’ truncate if > maxTokens (char-based estimation).

#### Data
- **Structures**: `Repository { q db.Querier, sqldb *sql.DB }`, `TreeNode { Message, Parent, Children, Depth }`, `EditMessageParams { SessionID, TargetID, Content, Role }`, `ForkSessionParams { OriginalSessionID, AgentName, Title, FromMessageID, NewContent }`
- **Flow**: SelectLeaf â†’ verify â†’ update. EditMessage â†’ validate â†’ tx â†’ create sibling â†’ update active. ForkSession â†’ copy branch â†’ new session â†’ add msg â†’ set active. BuildContext â†’ load branch â†’ convert â†’ truncate â†’ return `[]llm.Message`

### `llm/`

- **Flow**: Request â†’ marshal â†’ POST â†’ parse response â†’ return completion. Stream â†’ POST â†’ ParseSSEStream â†’ yield events â†’ parse Data
