### `internal`
GoPengAI framework internal packages — agent logic, API, config, DB, history, LLM, tools.

---

### `agent/`
Agent configs, registry, loader, engine, types.

#### Algorithm
Loader parses `.md` files with YAML frontmatter → builds `Agent` struct (name, system prompt, tools, permissions). Registry stores agents in-memory, concurrency-safe. Engine runs core async loop: load agent → build context → call LLM → execute tools → persist messages → publish events via EventBus. Supports abort via context cancellation. Adapters bridge to tools package interfaces (avoids circular imports).

#### Data
- **Structures**: `Engine { LLM, Tools, History, Agents, SQLDB, Querier, Config, EventBus, abortFuncs, idCounter }`, `Agent { Name, SystemPrompt, Model, Tools, ParentAgent, Permissions, ConfigPath, Description, Mode, Color, AgentPerms }`, `Message { Role, Content, ToolCalls, ToolCallID, Name }`, `ToolCall { ID, Name, Arguments }`, `Response { Content, Usage, StopReason, Error }`, `Usage { PromptTokens, CompletionTokens, TotalTokens }`
- **Flow**: Load `.md` → YAML frontmatter → parse → register → retrieve by name → Engine.Process() → async loop → events

### `api/`
HTTP handlers, routing, session CRUD, chat, OpenAI compatibility.

#### Algorithm
Session CRUD: create (UUID fallback), list, get (+messages), delete (cascade, wrapped in transaction). Chat flow: transaction (read, save user, update status) → LLM call → transaction (save assistant, update) → return response. Error handling: logs + 500/502 responses, sanitized (no leakage). Input size limit (100KB). OpenAI pass-through: `/v1/chat/completions` forwards to LLM. Routes: Go 1.22+ method patterns. EventBus fully implemented (pub/sub, heartbeat, close).

#### Data
- **Structures**: `Handler { LLM, DB, SQLDB, Config, Engine, EventBus }`, `EventBus { mu, global, sessions, closed, done }`, `ChatRequest { Model, Messages, Tools, ToolChoice }`, `CreateSessionRequest { Title, AgentName }`, `ChatResponse { SessionID, MessageID, Role, Content, Model, Usage, Error }`, `SessionView { ID, AgentName, Title, Status, MessageCount, CreatedAt, UpdatedAt, Messages }`, `MessageView { ID, Role, Content, AgentName, CreatedAt }`
- **Flow**: POST `/session` → create → return view. GET `/session/{id}` → load + messages → return view. POST `/session/{id}/message` → transaction → LLM → transaction → return response. POST `/v1/chat/completions` → forward → return raw. SSE: Subscribe → receive events → Unsubscribe.

### `config/`
JSON config loader, defaults, env overrides.

#### Algorithm
Read JSON → unmarshal → apply defaults → apply env overrides → return Config. Priority: JSON < defaults < env vars. Env vars parse int for port/iterations.

#### Data
- **Structures**: `Config { Server, LLM, AgentsDir, DataDir, DefaultAgent }`, `ServerConfig { Host, Port }`, `LLMConfig { Provider, BaseURL, APIKey, Model, MaxIterations }`
- **Flow**: file path → read → json.Unmarshal → applyDefaults → applyEnvOverrides → return *Config

---

### `db/`
SQLite connectivity, migrations, sqlc-generated query code.

#### Algorithm
Open SQLite (pure Go, ncruces driver), set pragmas (WAL, FK, cache, sync). Migrate with embedded Goose. sqlc generates `Queries`, `models`, `querier`, per-query `.sql.go`.

#### Data
- **Structures**: `Queries { DBTX, prepared stmts }`, DB models: `Agent`, `DelegationLog`, `Memory`, `Message`, `Session`
- **Flow**: Open → set pragmas → migrate → prepare queries → CRUD ops

---

### `history/`
Conversation history tree, message editing, context building.

#### Algorithm
**SelectLeaf**: Verify leaf, update active_leaf_id. **EditMessage**: Create sibling message (new branch), transaction, update active leaf. **ForkSession**: Copy branch up to `FromMessageID`, append new user message, create new session. **BuildContext**: Load active branch → convert to `llm.Message` → prepend system prompt → truncate if > maxTokens (char-based estimation).

#### Data
- **Structures**: `Repository { q db.Querier, sqldb *sql.DB }`, `TreeNode { Message, Parent, Children, Depth }`, `EditMessageParams { SessionID, TargetID, Content, Role }`, `ForkSessionParams { OriginalSessionID, AgentName, Title, FromMessageID, NewContent }`
- **Flow**: SelectLeaf → verify → update. EditMessage → validate → tx → create sibling → update active. ForkSession → copy branch → new session → add msg → set active. BuildContext → load branch → convert → truncate → return `[]llm.Message`

### `llm/`
LLM client, streaming, OpenAI-compatible API.

#### Algorithm
**Client**: HTTP POST to `/chat/completions` with Authorization. **Stream**: SSE parser, yield events, parse chunks. **Types**: Request/Response structs matching OpenAI API.

#### Data
- **Structures**: `Client { BaseURL, APIKey, Model, HTTPClient }`, `ChatCompletionRequest { Model, Messages, Tools, ToolChoice }`, `Message { Role, Content, ToolCalls, ToolCallID, Name }`, `ChatCompletionResponse { ID, Model, Choices, Usage }`, `SSEEvent { Event, Data }`
- **Flow**: Request → marshal → POST → parse response → return completion. Stream → POST → ParseSSEStream → yield events → parse Data

---

### `tools/`
Tool implementations — registry, web fetch, memory, delegation.

#### Algorithm
**Registry**: Concurrency-safe tool storage, `Register`/`Get`/`ToToolDefinitions`/`IsAllowed`. **WebFetchTool**: HTTP GET with User-Agent, 10s timeout, 50KB max, HTML stripping (script/style tags, entities, whitespace). **MemorySave/MemoryRecall**: DB operations scoped to agent name (via context keys `ContextKeyAgentName`, `ContextKeyQuerier`). **DelegateTool**: Sub-agent execution with 30s timeout, cycle detection (`ContextKeyVisitedAgents`), delegation logs (DB). Adapters avoid circular imports (`AgentLite`, `LLMClientLite`).

#### Data
- **Structures**: `Tool interface { Name, Description, Parameters, Execute }`, `Registry { tools map[string]Tool }`, `WebFetchTool { Client, MaxResponseBytes }`, `MemorySave{}`, `MemoryRecall{}`, `DelegateTool{}`, `ToolDefinition { Type, Function }`, `ToolFunction { Name, Description, Parameters }`
- **Flow**: Register tools → LLM requests definitions → Execute tool via context (agent name, querier, registry, LLM client) → return result string.
