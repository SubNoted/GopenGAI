### `internal`
GoPengAI framework internal packages ‚Äî agent logic, API, config, DB, history, LLM, tools.

---

### `agent/`
Agent configs, registry, loader, engine, types.

#### Algorithm
Loader parses `.md` files with YAML frontmatter ‚Üí builds `Agent` struct (name, system prompt, tools, permissions). Registry stores agents in-memory, concurrency-safe. Engine runs core async loop: load agent ‚Üí build context ‚Üí call LLM ‚Üí execute tools ‚Üí persist messages ‚Üí publish events via EventBus. Supports abort via context cancellation. Adapters bridge to tools package interfaces (avoids circular imports).

#### Data
- **Structures**: `Engine { LLM, Tools, History, Agents, Querier, Config, EventBus, abortFuncs, idCounter }`, `Agent { Name, SystemPrompt, Model, Tools, ParentAgent, Permissions, ConfigPath, Description, Mode, Color, AgentPerms }`, `Message { Role, Content, ToolCalls, ToolCallID, Name }`, `ToolCall { ID, Name, Arguments }`, `Response { Content, Usage, StopReason, Error }`, `Usage { PromptTokens, CompletionTokens, TotalTokens }`
- **Flow**: Load `.md` ‚Üí YAML frontmatter ‚Üí parse ‚Üí register ‚Üí retrieve by name ‚Üí Engine.Process() ‚Üí async loop ‚Üí events

### `api/`
HTTP handlers, routing, session CRUD, chat, SSE streaming, middleware, OpenAI compatibility.

#### Algorithm
Async message flow: atomic `UPDATE status='working' WHERE status='idle'` (DB-level concurrency guard) ‚Üí save user msg ‚Üí spawn engine goroutine ‚Üí return 202 ‚Üí results streamed via SSE. Session CRUD: create (UUID), list, get (+messages), delete (transaction). SSE: `writeSSE` formats events, sanitizes type+data, flushes; `streamSSE` shared impl for global/session streams. Error handling: sanitized (no `err.Error()` leakage), `encodeJSON` helper logs encode errors. Input size limit: 1MB MaxBytesReader on all body endpoints. Middleware: logging (method/path/status/duration via responseWriter), CORS (with Vary: Origin), recovery (panic-safe). Routes: Go 1.22+ method patterns, 22+ endpoints wired (session CRUD, agents, memory, SSE, branches, fork, abort, models). EventBus: pub/sub, heartbeat (15s), session scoping, slow-consumer drop, clean close. Handler fields include Wg *sync.WaitGroup for graceful goroutine tracking.

#### Data
- **Structures**: `Handler { LLM, DB, SQLDB, Config, Engine, EventBus, History, Wg }`, `EventBus { mu, global, sessions, closed, done }`, `ChatRequest { Model, Messages, Tools, ToolChoice }`, `CreateSessionRequest { Title, AgentName }`, `ChatResponse { SessionID, MessageID, Role, Content, Model, Usage, Error }`, `SessionView { ID, AgentName, Title, Status, MessageCount, CreatedAt, UpdatedAt, Messages }`, `MessageView { ID, Role, Content, AgentName, CreatedAt }`, `SSEEvent { Type, Properties }`
- **Flow**: POST `/session` ‚Üí create ‚Üí view. POST `/session/{id}/message` ‚Üí atomic status claim ‚Üí 202 ‚Üí engine goroutine ‚Üí SSE events. GET `/events` ‚Üí global SSE stream. GET `/session/{id}/events` ‚Üí session SSE stream. Middleware chain: Recovery ‚Üí CORS ‚Üí Logging ‚Üí mux.

### `config/`
JSON config loader, defaults, env overrides.

#### Algorithm
Read JSON ‚Üí unmarshal ‚Üí apply defaults ‚Üí apply env overrides ‚Üí return Config. Priority: JSON < defaults < env vars. Env vars parse int for port/iterations.

#### Data
- **Structures**: `Config { Server, LLM, AgentsDir, DataDir, DefaultAgent }`, `ServerConfig { Host, Port }`, `LLMConfig { Provider, BaseURL, APIKey, Model, MaxIterations }`
- **Flow**: file path ‚Üí read ‚Üí json.Unmarshal ‚Üí applyDefaults ‚Üí applyEnvOverrides ‚Üí return *Config

### `db/`
SQLite connectivity, migrations, sqlc-generated query code.

#### Algorithm
Open SQLite (pure Go, ncruces driver), set pragmas (WAL, FK, cache, sync). Migrate with embedded Goose. sqlc generates `Queries`, `models`, `querier`, per-query `.sql.go`.

#### Data
- **Structures**: `Queries { DBTX, prepared stmts }`, DB models: `Agent`, `DelegationLog`, `Memory`, `Message`, `Session`
- **Flow**: Open ‚Üí set pragmas ‚Üí migrate ‚Üí prepare queries ‚Üí CRUD ops

### `history/`
Conversation history tree, message editing, context building.

#### Algorithm
**SelectLeaf**: Verify leaf, update active_leaf_id. **EditMessage**: Create sibling message (new branch), transaction, update active leaf. **ForkSession**: Copy branch up to `FromMessageID`, append new user message, create new session. **BuildContext**: Load active branch ‚Üí convert to `llm.Message` ‚Üí prepend system prompt ‚Üí truncate if > maxTokens (char-based estimation).

#### Data
- **Structures**: `Repository { q db.Querier, sqldb *sql.DB }`, `TreeNode { Message, Parent, Children, Depth }`, `EditMessageParams { SessionID, TargetID, Content, Role }`, `ForkSessionParams { OriginalSessionID, AgentName, Title, FromMessageID, NewContent }`
- **Flow**: SelectLeaf ‚Üí verify ‚Üí update. EditMessage ‚Üí validate ‚Üí tx ‚Üí create sibling ‚Üí update active. ForkSession ‚Üí copy branch ‚Üí new session ‚Üí add msg ‚Üí set active. BuildContext ‚Üí load branch ‚Üí convert ‚Üí truncate ‚Üí return `[]llm.Message`

### `llm/`
LLM client, streaming, OpenAI-compatible API.

#### Algorithm
**Client**: HTTP POST to `/chat/completions` with Authorization. **Stream**: SSE parser, yield events, parse chunks. **Types**: Request/Response structs matching OpenAI API.

#### Data
- **Structures**: `Client { BaseURL, APIKey, Model, HTTPClient }`, `ChatCompletionRequest { Model, Messages, Tools, ToolChoice }`, `Message { Role, Content, ToolCalls, ToolCallID, Name }`, `ChatCompletionResponse { ID, Model, Choices, Usage }`, `SSEEvent { Event, Data }`
- **Flow**: Request ‚Üí marshal ‚Üí POST ‚Üí parse response ‚Üí return completion. Stream ‚Üí POST ‚Üí ParseSSEStream ‚Üí yield events ‚Üí parse Data

### `tools/`
Tool implementations ‚Äî registry, web fetch, memory, delegation.

#### Algorithm
**Registry**: Concurrency-safe tool storage, `Register`/`Get`/`ToToolDefinitions`/`IsAllowed`. **WebFetchTool**: HTTP GET with User-Agent, 10s timeout, 50KB max, HTML stripping (script/style tags, entities, whitespace). **MemorySave/MemoryRecall**: DB operations scoped to agent name (via context keys `ContextKeyAgentName`, `ContextKeyQuerier`). **DelegateTool**: Sub-agent execution with 30s timeout, cycle detection (`ContextKeyVisitedAgents`), delegation logs (DB). Adapters avoid circular imports (`AgentLite`, `LLMClientLite`).

#### Data
- **Structures**: `Tool interface { Name, Description, Parameters, Execute }`, `Registry { tools map[string]Tool }`, `WebFetchTool { Client, MaxResponseBytes }`, `MemorySave{}`, `MemoryRecall{}`, `DelegateTool{}`, `ToolDefinition { Type, Function }`, `ToolFunction { Name, Description, Parameters }`
- **Flow**: Register tools ‚Üí LLM requests definitions ‚Üí Execute tool via context (agent name, querier, registry, LLM client) ‚Üí return result string.
