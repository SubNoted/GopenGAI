### `handler.go`
HTTP request handlers with session management, chat, transactions, error handling.

#### Algorithm
**Session CRUD**: Create sessions with UUID fallback ID, list all, get by ID + messages, delete (cascade messages).  
**Chat message flow**: Read session → save user message (transaction) → update session status/active leaf → call LLM → save assistant response (transaction) → return. Uses transaction for race prevention during concurrent requests. LLM call outside transaction to avoid DB lock.  
**Error handling**: `internalError` logs + sends 500. `saveLLMErrorAndRespond` saves error message to DB, returns 502.  
**OpenAI pass-through**: `HandleChatCompletion` accepts `/v1/chat/completions`, forwards to LLM, no history persistence.  
**UUID generation**: `newID()` uses `crypto/rand` hex string, fallback to timestamp-based on failure.

#### Key Functions
- `HandleCreateSession`: Decode request → default agent/title → insert session → return 201  
- `HandleListSessions`: List all sessions → convert to views → JSON encode  
- `HandleGetSession`: Get by ID + messages → return view with messages  
- `HandleDeleteSession`: Delete messages first (FK) → delete session → return status  
- `HandleChatMessage`: Transaction (read + save user + update status) → LLM call → transaction (save assistant + update) → return response  
- `HandleHealth`: Simple JSON health check  
- `HandleChatCompletion`: Decode OpenAI request → forward to LLM → return raw response  

#### Data
- **Structures**:  
  - `Handler { LLM *llm.Client, DB *db.Queries, SQLDB *sql.DB, Config *config.Config }`  
  - `ChatRequest { Model, Messages []llm.Message, Tools []llm.ToolDefinition, ToolChoice json.RawMessage }`  
  - `CreateSessionRequest { Title, AgentName }`  
  - `ChatResponse { SessionID, MessageID, Role, Content, Model, Usage *llm.Usage, Error }`  
  - `SessionView { ID, AgentName, Title, Status, MessageCount, CreatedAt, UpdatedAt, Messages []MessageView }`  
  - `MessageView { ID, Role, Content, AgentName, CreatedAt }`  
- **Types**: UUID string (hex format), timestamps in UnixMilli  
- **Flow**: POST `/session` → create → return view. GET `/session/{id}` → load session + messages → return view. POST `/session/{id}/message` → transaction → LLM → transaction → return response. POST `/v1/chat/completions` → forward to LLM → return raw.

#### External Dependencies
- `./internal/config`
- `./internal/db`
- `./internal/llm`

### `routes.go`
Register routes on `http.ServeMux` using Go 1.22+ method routing patterns.

#### Algorithm
Wires all handler methods to serve mux with explicit method + path patterns. No middleware yet.

#### Key Functions
- `RegisterRoutes(mux *http.ServeMux, h *Handler)`:  
  - `GET /health` → `HandleHealth`  
  - `POST /session` → `HandleCreateSession`  
  - `GET /session` → `HandleListSessions`  
  - `GET /session/{id}` → `HandleGetSession`  
  - `DELETE /session/{id}` → `HandleDeleteSession`  
  - `POST /session/{id}/message` → `HandleChatMessage`  
  - `POST /v1/chat/completions` → `HandleChatCompletion`

#### Data
- **Flow**: Serve mux → register pattern → handler method
