### `events.go`
EventBus — concurrency-safe pub/sub for SSE events with heartbeat, session scoping, slow-consumer protection, clean shutdown via done channel.

#### Algorithm
- `EventBus` struct with `sync.RWMutex`, `global []chan SSEEvent`, `sessions map[string][]chan SSEEvent`, `done chan struct{}`
- `SSEEvent` struct: `Type string`, `Properties interface{}`
- `NewEventBus()` creates empty bus with done channel + starts heartbeat goroutine (15s ticker)
- `Subscribe(sessionID)` — returns new buffered channel (buffer=64); sessionID="" = global listener
- `Unsubscribe(sessionID, ch)` — removes channel from bus and closes it; safe to call multiple times
- `PublishGlobal(eventType, props)` / `PublishSession(sessionID, eventType, props)` — primitive-typed methods implementing `agent.EventBus` interface; wrap internal `SSEEvent` structs
- Slow listener protection: `select default` drops events when channel buffer full
- `Subscribe()` after `Close()` — returns already-closed channel (caller's `range ch` exits immediately, no goroutine leak)
- `Close()` — closes done channel (stops heartbeat), closes all listener channels, nil-ifies slices/maps, no-op if already closed
- Heartbeat goroutine uses `select` on `done` channel + ticker.C (no TOCTOU gap between closed-check and publish)

#### Exports
- `EventBus` struct (implements `agent.EventBus` interface)
- `SSEEvent` struct

### `sse.go`
Server-Sent Events (SSE) — real-time event streaming for async agent engine. Global (`/events`) and session-specific (`/session/{id}/events`) endpoints.

#### Algorithm
- **`writeSSE(w, event)`**: Marshals `event.Properties` to JSON, sanitizes `event.Type` and data (strips `\n`, `\r`), writes `event: <type>\ndata: <json>\n\n`, flushes immediately.
- **`sseHeaders(w)`**: Sets `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`, `X-Accel-Buffering: no`.
- **`HandleGlobalSSE`**: GET `/events`, delegates to `streamSSE` with empty sessionID (global listener).
- **`HandleSessionSSE`**: GET `/session/{id}/events`, validates path param, delegates to `streamSSE`.
- **`streamSSE`** (shared impl):
  - Checks `http.Flusher` support.
  - Calls `sseHeaders`.
  - Writes `: connected\n\n` (initial heartbeat).
  - Subscribes to `EventBus`.
  - Loops `select` on `ctx.Done()` (client disconnect) or event channel.
  - Calls `writeSSE` for each event.
  - Logs errors per session/global.

#### Data
- **Flow**: Client connects → set headers → subscribe EventBus → loop select on events/ctx.Done → write SSE formatted event → flush → continue.

#### Exports
- `HandleGlobalSSE`
- `HandleSessionSSE`

### `handler.go`
HTTP request handlers — session CRUD, async chat (202 Accepted), branch management, agents, memory, SSE integration.

#### Algorithm
- **Async Flow**: `POST /session/{id}/message` returns 202 Accepted, spawns engine goroutine with 5-minute timeout, `sync.WaitGroup` tracking.
- **Concurrent Prevention**: Atomic `UPDATE sessions SET status='working' WHERE id=? AND status='idle'` with `RowsAffected` check (DB-level mutual exclusion).
- **Panic Recovery**: Defer `recover()` in async goroutine, logs stack trace, always resets DB status to "idle".
- **Input Limits**: `http.MaxBytesReader` (1MB) on all body-reading endpoints. `maxContentLength` (100KB) for user messages.
- **Error Sanitization**: `encodeJSON()` logs json.Encode errors. Fixed error strings for client (no `err.Error()` leak). `internalError` logs server-side, returns generic 500.
- **Session CRUD**: Transactional delete (messages + session). UUID generation with crypto/rand or timestamp fallback.
- **OpenAI Pass-through**: `HandleChatCompletion` forwards to LLM, returns raw response, no history persistence.
- **History/State**: `History` field added to Handler. `ActiveLeafID` updated by engine.

#### Key Functions
- `HandleCreateSession`: Decode request → default agent/title → insert session → return 201
- `HandleListSessions`: List all → convert to views → JSON encode
- `HandleGetSession`: Get by ID + messages → return view with messages
- `HandleDeleteSession`: BeginTx → delete messages → delete session → commit → return status
- `HandleChatMessage`: Parse body → validate → atomic status claim → return 202 → spawn async engine goroutine
- `HandleUpdateSession`: PATCH title → update DB → return view
- `HandleGetSessionMessages`: Get active branch messages via History.GetActiveBranch
- `HandleListBranches`: List all leaf messages via History.GetAllLeaves
- `HandleForkSession`: Fork at message → create new session → return 201
- `HandleSelectBranch`: Set active leaf → update ActiveLeafID
- `HandleEditMessage`: Patch message → create new sibling branch → return 201
- `HandleAbortSession`: Call Engine.Abort(sessionID) → return status
- `HandleListAgents` / `HandleGetAgent`: List/fetch agent registry
- `HandleListMemory` / `HandleGetMemory`: List/fetch memory facts by agent
- `HandleListModels`: List agents as OpenAI-compatible models
- `HandleChatCompletion`: Forward OpenAI request to LLM, return raw response
- `HandleHealth`: Simple JSON health check

#### Data
- **Structures**:
  - `Handler { LLM, DB, SQLDB, Config, Engine, EventBus, History, Wg *sync.WaitGroup }`
  - `ChatRequest { Model, Messages, Tools, ToolChoice }`
  - `CreateSessionRequest { Title, AgentName }`
  - `ForkRequest { MessageID, Content, Title, AgentName }`
  - `SelectBranchRequest { LeafID }`
  - `EditMessageRequest { SessionID, Content, Role }`
  - `SessionView { ID, AgentName, Title, Status, MessageCount, CreatedAt, UpdatedAt, Messages }`
  - `MessageView { ID, Role, Content, AgentName, CreatedAt }`
  - `AgentView { Name, Model, Tools, ParentAgent, Description, Mode }`
  - `MemoryView { ID, AgentName, Key, Value, Category, CreatedAt, UpdatedAt }`
  - `ModelView { ID, Object, Created, OwnedBy }`
- **Types**: UUID string (hex format), timestamps in UnixMilli.
- **Flow**:
  - POST `/session` → create → return view.
  - POST `/session/{id}/message` → atomic claim → 202 Accepted → async engine process → SSE events.
  - GET `/session/{id}/events` → SSE stream.
  - PATCH `/session/{id}` → update title → return view.
  - POST `/session/{id}/fork` → duplicate history branch → return new session.
  - PUT `/session/{id}/branch` → set active leaf.
  - PATCH `/messages/{id}` → edit message → create branch.

### `middleware.go`
Standard HTTP middleware — logging, CORS, panic recovery, bearer-token authentication.

#### Algorithm
- **`responseWriter`**: Wraps `http.ResponseWriter`, captures status code, forwards `Flusher`, `Hijacker`, `Pusher`.
- **`LoggingMiddleware`**: Logs `METHOD /path remote_addr status duration` for every request.
- **`CORSMiddleware`**: Sets `Access-Control-Allow-Origin: *`, allows GET/POST/PUT/PATCH/DELETE/OPTIONS, accepts `Content-Type`, `Authorization`, `X-Requested-With`. Returns 204 on OPTIONS. Sets `Vary: Origin`.
- **`RecoveryMiddleware`**: `defer recover()`, logs stack trace, returns 500 (if headers not committed).
- **`AuthMiddleware(apiKey)`**: If apiKey is empty → pass-through. `/health` always allowed. Checks `Authorization: Bearer <key>`. Returns 401 JSON on mismatch.
- **`ApplyMiddleware`**: Chains middleware outer→inner (first in list runs first).

#### Exports
- `AuthMiddleware(apiKey string) func(http.Handler) http.Handler`
- `LoggingMiddleware(next http.Handler) http.Handler`
- `CORSMiddleware(next http.Handler) http.Handler`
- `RecoveryMiddleware(next http.Handler) http.Handler`
- `ApplyMiddleware(h http.Handler, mw ...func(http.Handler) http.Handler) http.Handler`

### `routes.go`
Register routes on `http.ServeMux` using Go 1.22+ method routing patterns.

#### Algorithm
Wires all handler methods to serve mux with explicit method + path patterns. Middleware applied in `cmd/api/main.go`.

#### Key Functions
- `RegisterRoutes(mux *http.ServeMux, h *Handler)`:
  - `GET /health` → `HandleHealth`
  - `POST /session` → `HandleCreateSession`
  - `GET /session` → `HandleListSessions`
  - `GET /session/{id}` → `HandleGetSession`
  - `PATCH /session/{id}` → `HandleUpdateSession`
  - `DELETE /session/{id}` → `HandleDeleteSession`
  - `GET /session/{id}/messages` → `HandleGetSessionMessages`
  - `GET /session/{id}/branches` → `HandleListBranches`
  - `POST /session/{id}/fork` → `HandleForkSession`
  - `PUT /session/{id}/branch` → `HandleSelectBranch`
  - `POST /session/{id}/abort` → `HandleAbortSession`
  - `POST /session/{id}/message` → `HandleChatMessage` (async 202)
  - `GET /events` → `HandleGlobalSSE`
  - `GET /session/{id}/events` → `HandleSessionSSE`
  - `PATCH /messages/{id}` → `HandleEditMessage`
  - `GET /agents` → `HandleListAgents`
