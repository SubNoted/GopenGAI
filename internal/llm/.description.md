### `internal/llm`
LLM client & streaming logic — OpenAI-compatible API.

---

### `types.go`
LLM type definitions — request/response structs with tool calling support.

#### Algorithm
Define struct types matching OpenAI API format. Request: Model, Messages, Tools, ToolChoice. Response: ID, Model, Choices, Usage. Streaming: incremental deltas per SSE chunk.

#### Data
- **Structures:** ChatCompletionRequest, Message, ToolDefinition, ToolFunction, ToolCall, FunctionCall, ChatCompletionResponse, Choice, MessageResponse, Usage, APIError, StreamCompletionResponse, StreamChoice, StreamDelta
- **Types:** ChatCompletionRequest {Model, Messages, Tools, ToolChoice}, Message {Role, Content, ToolCalls, ToolCallID, Name}, ToolCall {ID, Type, Function}, FunctionCall {Name, Arguments}, StreamDelta {Role, Content, ToolCalls}
- **Flow:** Request built → JSON marshal → send to /chat/completions → unmarshal response → return struct

---

### `client.go`
LLM client — HTTP POST to /chat/completions endpoint.

#### Algorithm
NewClient/NewClientFromConfig creates Client with BaseURL, APIKey, Model, HTTPClient. ChatCompletion: marshal request, POST to BaseURL+"/chat/completions" with Authorization header, parse response, return completion or LLMError. Error handling: if status != 200, try parse APIError, else raw body.

#### Data
- **Structures:** Client {BaseURL, APIKey, Model, HTTPClient}, LLMError {StatusCode, Message, APIError}
- **Types:** Client struct fields
- **Flow:** req → marshal → POST → read body → if ok unmarshal → return response; if error → parse APIError or raw body → return LLMError

---

### `stream.go`
SSE streaming parser & streaming completion support.

#### Algorithm
ParseSSEStream: scan lines, ignore comments, parse "event:" and "data:" lines, yield SSEEvent on empty line. StreamCompletion: marshal req, POST with Accept:text/event-stream header, pipe resp.Body to ParseSSEStream, yield events to channel, send error on err channel. ParseStreamData: unmarshal SSE data JSON to StreamCompletionResponse. IsStreamDone: check any Choice has FinishReason.

#### Data
- **Structures:** SSEEvent {Event, Data}
- **Types:** SSEEvent struct
- **Flow:** stream req → POST → ParseSSEStream → yield SSEEvent → consumer parses Data as StreamCompletionResponse → IsStreamDone checks finish_reason
