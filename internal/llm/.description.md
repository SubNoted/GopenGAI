### `internal/llm`
Directory containing LLM (Large Language Model) client and streaming logic.

---

### `types.go`
LLM type definitions — OpenAI-compatible request/response structs with tool calling support.

#### Exports
- `ChatCompletionRequest` struct (`Model`, `Messages`, `Tools`, `ToolChoice` with JSON tags matching OpenAI API)
- `Message` struct (`Role`, `Content`, `ToolCalls`, `ToolCallID`, `Name`)
- `ToolDefinition` struct (`Type`, `Function`) — describes a callable tool to the LLM
- `ToolFunction` struct (`Name`, `Description`, `Parameters`) — JSON Schema for a tool
- `ToolCall` struct (`ID`, `Type`, `Function`) — represents an LLM-requested function call
- `FunctionCall` struct (`Name`, `Arguments`) — the name and JSON arguments of a tool invocation
- `ChatCompletionResponse` struct (`ID`, `Object`, `Created`, `Model`, `Choices`, `Usage`, `Error`)
- `Choice` struct (`Index`, `Message` as `MessageResponse`, `FinishReason`)
- `MessageResponse` struct (`Role`, `Content`, `ToolCalls`) — assistant message from LLM
- `Usage` struct (`PromptTokens`, `CompletionTokens`, `TotalTokens`)
- `APIError` struct (`Message`, `Type`, `Code`)
- `StreamCompletionResponse` struct — single SSE chunk for streaming
- `StreamChoice` struct (`Index`, `Delta`, `FinishReason`)
- `StreamDelta` struct (`Role`, `Content`, `ToolCalls`)

### `client.go`
LLM client implementation — HTTP POST/GET to OpenAI-compatible `/chat/completions` endpoint.

#### Exports
- `Client` struct with `BaseURL`, `APIKey`, `Model`, `HTTPClient` fields
- `NewClient(baseURL, apiKey, model string) *Client`
- `NewClientFromConfig(cfg config.LLMConfig) *Client`
- `(c *Client) ChatCompletion(ctx context.Context, req *ChatCompletionRequest) (*ChatCompletionResponse, error)` — sends request with optional tools/tool_choice
- `LLMError` struct (`StatusCode`, `Message`, `APIError`) — structured error type

### `stream.go`
LLM stream handling — SSE parsing and streaming completion support.

#### Exports
- `SSEEvent` struct (`Event`, `Data`)
- `ParseSSEStream(ctx, io.Reader, chan<- SSEEvent) error` — low-level SSE parser
- `(c *Client) StreamCompletion(ctx, *ChatCompletionRequest) (<-chan SSEEvent, <-chan error)` — streaming completion
- `ParseStreamData(data string) (*StreamCompletionResponse, error)` — unmarshal SSE data JSON
- `IsStreamDone(chunk *StreamCompletionResponse) bool` — check for finish_reason
