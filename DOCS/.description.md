### `DOCS`
Project documentation directory — contains architecture diagrams and API reference.

#### Algorithm
Stores Mermaid-based diagrams and markdown API docs. Diagrams cover: container views, agent loop, message flow, history tree, database ERD, package structure, REST API, SSE events, dev responsibilities (Who Wrote What). Files ordered numerically for logical progression.

#### Data
- **Structures:** Directory tree with `.description.md` + `diagrams/` subdirectory
- **Types:** Markdown files with Mermaid code blocks, API schemas in JSON + Go types
- **Flow:** Read diagram files → understand architecture → implement features

### `diagrams/02-agent-loop.md`
Agent loop core engine logic.

#### Algorithm
Processes single request: parse API mode → load session/agent → build context → call LLM → handle response. Tool calls loop back: web_fetch, memory_save, memory_recall, delegate_agent. Max iterations prevents infinite loops.

#### Data
- **Structures:** Flowchart nodes for parse, load, build, call, execute
- **Types:** Response type enum (Stop, Tool call)
- **Flow:** Incoming message → Parse → Load → Build → Call LLM → Parse → Tool/Stop

### `diagrams/04-history-tree.md`
Branching conversation model.

#### Algorithm
Sessions use tree structure: messages have parent_id (NULL for root). Active leaf points to current branch tip. Edits create new branches (immutable history). Multiple leaves per session.

#### Data
- **Structures:** Message tree nodes with role, content, parent_id
- **Types:** Session {id, active_leaf_id}, Message {id, role, parent_id}
- **Flow:** Root → child messages → active leaf. Edit → new branch from parent.

### `diagrams/05-database-erd.md`
SQLite schema for sessions, messages, agents, memory, delegation.

#### Algorithm
OpenCode-compatible base extended with tree structure. Sessions contain active_leaf_id for branch cursor. Messages form tree via parent_id. Memory is key-value per agent. Delegation logs track sub-agent spawns.

#### Data
- **Structures:** SESSIONS, MESSAGES, AGENTS, MEMORY, DELEGATION_LOGS tables
- **Types:** UUID PKs, Unix ms timestamps, JSON parts/content
- **Flow:** Session → Messages tree → Agent configs → Memory facts → Delegation logs

### `diagrams/08-sse-events.md`
Server-Sent Events for real-time streaming.

#### Algorithm
Global stream for system events. Session stream for message updates. Event bus pub/sub in-memory. Heartbeat every 15s. Client reconnection with Last-Event-ID.

#### Data
- **Structures:** EventBus {global, sessions map}, SSEEvent {type, properties}
- **Types:** Event types: session.status, message.part.*, message.complete, heartbeat
- **Flow:** Engine publishes → EventBus → SSE writer → Client

### `diagrams/09-message-flow.md`
Async agent processing message flow.

#### Algorithm
POST returns 202, spawns goroutine. Save user message → load agent → build context → LLM call → tool exec → save results → stream SSE → update active leaf. Max iterations limit. Abort via context cancel.

#### Data
- **Structures:** Flowchart for request lifecycle, state machine for session status
- **Types:** Session status: idle, working; Tool permission: allow/deny
- **Flow:** API 202 → goroutine → engine loop → SSE events → DB save → idle
