### `01-разделение-работ.md`
Документация разделения работы между разработчиками (Adile vs Subnoted).

#### Algorithm
Описывает логическое разделение системы на слои. Использует Mermaid flowchart для визуализации ответственности: Adile (API/CLI/DB/Infra) и Subnoted (Engine/History/Tools/LLM). Ключевые функции: перечисление компонентов, кто делал, как части соединяются (контракт), и доказательства разделения для защиты. Стратегия: четкое разграничение слоев для независимой разработки и тестирования.

#### Data
- **Структуры:** Слои системы (Adile, Subnoted, Shared), компоненты (cmd/api, internal/agent, internal/db и т.д.).
- **Типы:** Разработчики (Adile, Subnoted), роли (Front/Infra, Brain).
- **Поток:** Запрос -> HTTP Handler (Adile) -> Agent Engine (Subnoted) -> DB (Adile) -> SSE (Adile) -> Пользователь.

### `НАРАБОТКИ_ПЗ.md`
Полный документ курсовой работы: проектирование микросервиса для интеграции LLM.

#### Algorithm
Описывает весь проект: постановка задачи, анализ требований, теоретическое описание (Event-Driven Architecture, Tree-based History, Agent-Tool Pattern), архитектуру системы, реализацию. Включает листинги кода (заполнители) и распределение страниц. Стратегия: документирование всего lifecycle проекта от требований до реализации и тестирования.

#### Data
- **Структуры:** Система GoPengAI (микросервис, агенты, инструменты), БД SQLite (5 таблиц: sessions, messages, branches, configs, memory).
- **Типы:** Конфигурация (JSON/ENV), Агенты (Markdown + YAML frontmatter).
- **Поток:** Пользователь -> HTTP API -> Agent Engine -> LLM -> SSE -> Пользователь.
