# Инструменты сборки и разработки

## Makefile

| Команда | Действие |
|---------|----------|
| `make build` | `go build ./cmd/api/ ./cmd/cli/` |
| `make run` | `go run ./cmd/api/` (запуск сервера) |
| `make test` | `go test ./... -v` |
| `make lint` | `go vet ./...` |
| `make fmt` | `go fmt ./...` |
| `make tidy` | `go mod tidy` |
| `make sqlc-generate` | `sqlc generate` |
| `make clean` | Удаляет бинарники и `.gopengai/` |

## Зависимости (go.mod)

```
module gopengai

go 1.26.1

require (
    github.com/ncruces/go-sqlite3 v0.34.2      // SQLite драйвер (Pure Go, WASM)
    github.com/pressly/goose/v3 v3.27.1         // Миграции
    github.com/spf13/cobra v1.9.1               // CLI-фреймворк
)
```

Транзитивные зависимости:
- `ncruces/go-sqlite3-wasm` — WASM-прослойка SQLite
- `ncruces/julianday` — конвертация дат (нужна sqlite3)
- `spf13/pflag` — парсинг флагов (нужен Cobra)
- `pressly/goose` → `mfridman/interpolate`, `sethvargo/go-retry`, `golang.org/x/sync`

## Команды для разработки

```bash
# Сборка
go build ./cmd/api/   # → бинарник ./api
go build ./cmd/cli/   # → бинарник ./cli

# Проверка
go vet ./...

# Запуск сервера
./api
# или
make run

# Использование CLI
./cli chat "Привет"
./cli session list

# Регенерация sqlc (если меняли SQL)
sqlc generate
# или
make sqlc-generate

# Чистка модулей
go mod tidy

# Полный цикл
make clean && make build && make lint
```

## Важные замечания для разработки

### 1. sqlc требует go.mod
Перед `sqlc generate` нужно, чтобы `go.mod` существовал и все зависимости были скачаны (`go mod tidy`).

### 2. Goose миграции встроены в бинарник
Миграции лежат в `internal/db/migrations/` и встраиваются через `//go:embed migrations/*.sql`. Изменения в миграциях требуют пересборки бинарника.

### 3. Никаких CGo зависимостей
Весь проект компилируется без CGo благодаря `ncruces/go-sqlite3`. Это значит:
- `CGO_ENABLED=0 go build ./cmd/api/` работает
- Кросскомпиляция (linux → windows/mac) работает без дополнительных инструментов

### 4. Дата в формате Unix Millis
Все временные метки в БД — `INTEGER` с Unix миллисекундами (`time.Now().UnixMilli()`). Триггеры Goose используют `strftime('%s', 'now') * 1000`.

### 5. Go 1.22+ routing
HTTP-роутер использует синтаксис `"METHOD /path/{param}"`, который появился в Go 1.22. Для доступа к параметру: `r.PathValue("id")`.
