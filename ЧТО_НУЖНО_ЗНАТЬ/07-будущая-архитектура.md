## Фаза 4: Древовидная история (internal/history)

### Что будет

Вместо линейного списка сообщений — дерево с ветвлением.

```
Корень (root message)
├── "Расскажи про Go"
│   ├── "Go — это язык от Google..."
│   │   └── "А чем он лучше Python?"  ← активная ветка
│   └── "Ты не прав, Go — это..."     ← альтернативная ветка (форк)
└── "Привет"
    └── "Здравствуйте!"
```

### Рекурсивный CTE (уже в SQL!)

```sql
WITH RECURSIVE branch AS (
    SELECT * FROM messages WHERE messages.id = ?
    UNION ALL
    SELECT m.* FROM messages m
    INNER JOIN branch b ON m.id = b.parent_id
)
SELECT * FROM branch ORDER BY created_at ASC;
```

Этот запрос возвращает весь путь от любого сообщения до корня.

## Фаза 5: Инструменты (internal/tools)

### Интерфейс Tool

```go
type Tool interface {
    Name() string
    Description() string
    Parameters() json.RawMessage  // JSON Schema
    Execute(ctx context.Context, args json.RawMessage) (string, error)
}
```

### Планируемые инструменты

#### 1. web_fetch — загрузка веб-страниц
- HTTP GET с User-Agent
- timeout 10 секунд
- max response size 50KB
- Извлечение текста (без HTML)

#### 2. memory — долговременная память
- `memory_save(key, value, category)` — сохранить факт
- `memory_recall(key)` — прочитать факт
- Данные привязаны к имени агента
- Хранятся в таблице SQLite `memory`

#### 3. delegate — делегирование под-агенту
- Вызов другого агента из реестра
- Передача задачи текстом
- 30 секунд таймаут
- Защита от циклов (visited set имён агентов)
- Логирование в `delegation_logs`

## Фаза 6: Agent Engine (internal/agent/engine.go)

### Core Loop (упрощённо)

```
1. Получить сообщение пользователя
2. Загрузить агента из Registry
3. Построить контекст: system prompt + история + новое сообщение
4. LOOP:
   a. Вызвать LLM с инструментами
   b. Если ответ = текст → сохранить, вернуть
   c. Если ответ = tool_calls:
      - Для каждого tool call:
        - Проверить права (permissions)
        - Выполнить инструмент
        - Сохранить результат в историю
      - Продолжить LOOP (LLM получит результаты инструментов)
   d. Если итераций > max_iterations → прервать
5. Обновить active_leaf_id
6. Опубликовать события через EventBus
```

### Events (события для SSE)

| Событие | Описание |
|---------|----------|
| `session.status` | Статус сессии изменился (idle → working → idle) |
| `message.part.added` | Начало нового чанка текста |
| `message.part.updated` | Обновление чанка (для потокового вывода) |
| `message.complete` | Сообщение ассистента полностью сгенерировано |
| `message.tool.started` | Инструмент начал выполнение |
| `message.tool.completed` | Инструмент завершил выполнение |
| `message.tool.error` | Ошибка выполнения инструмента |
| `message.error` | Общая ошибка |

## Фаза 7: Асинхронный API + SSE

### Новый паттерн POST /session/{id}/message

```
ТЕКУЩИЙ (синхронный):
  POST /session/{id}/message → 200 {ответ JSON}

БУДУЩИЙ (асинхронный):
  POST /session/{id}/message → 202 Accepted
  Клиент подписан на SSE: GET /session/{id}/events
  SSE-события: message.part.added → ... → message.complete
```

### EventBus

```go
type EventBus struct {
    global   []chan SSEEvent          // глобальные подписчики
    sessions map[string][]chan SSEEvent  // подписчики на сессию
}

type SSEEvent struct {
    Type       string
    Properties interface{}
}

func (eb *EventBus) Subscribe(sessionID string) <-chan SSEEvent
func (eb *EventBus) Unsubscribe(sessionID string, ch <-chan SSEEvent)
func (eb *EventBus) PublishGlobal(event SSEEvent)
func (eb *EventBus) PublishSession(sessionID string, event SSEEvent)
```

SSE формат:
```
event: message.part.added
data: {"content":"Hello","index":0}

event: message.tool.started
data: {"tool":"web_fetch","args":{"url":"..."}}

event: message.complete
data: {"message_id":"abc","content":"Hello world!","usage":{...}}
```

## Риски архитектуры

| Риск | Серьёзность | Что делаем |
|------|------------|------------|
| SQLite не выдержит много запросов | Средняя | WAL + MaxOpenConns=1 + транзакции |
| sqlc код устареет при смене схемы | Низкая | Регенерировать при изменениях миграций |
| Утечка SSE-подписчиков (забыли каналы) | Средняя | Unsubscribe при disconnect, периодическая чистка |
| Разные форматы Tool Calling у провайдеров | Высокая | Абстракция LLM-клиента, тесты рано |
| Бесконечный цикл агента (LLM вызывает инструменты постоянно) | Средняя | max_iterations из конфига + таймаут |
| Рекурсивная делегация (агент → себя) | Средняя | visited set имён агентов |
| Переполнение контекстного окна | Средняя | Обрезка истории, суммаризация старых сообщений |
| Утечка горутин при выключении сервера | Средняя | Context cancellation + WaitGroup |
| WASM-прослойка ncruces медленная | Низкая | Бенчмарки; fallback на mattn/go-sqlite3 |
