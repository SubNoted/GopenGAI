### `api/`
API server entry point.

#### `main.go`
Entry point ÔÇö parses `--config` and `--port` flags, loads config, opens DB, runs migrations, init queries, creates LLM client, initializes agent/tool registries, creates history repo + event bus + engine, builds handler, registers routes, logs endpoints, starts HTTP server with full graceful shutdown (no os.Exit).

**Algorithm**
Parse flags: `--config` (default `"gopengai.json"`) for config path, `--port` (default `0`, meaning use config value) to override HTTP port. Load config Ôćĺ open DB Ôćĺ migrate Ôćĺ init queries Ôćĺ create LLM client Ôćĺ init agent registry Ôćĺ init tool registry (4 tools) Ôćĺ create history repo Ôćĺ create event bus Ôćĺ create engine (no longer needs SQLDB) Ôćĺ build handler (includes Engine, EventBus, History, Wg) Ôćĺ register routes (22+ endpoints) Ôćĺ start HTTP server in goroutine, send error to buffered `serverErr` chan Ôćĺ select on `quit`+`serverErr` Ôćĺ graceful shutdown: `SetKeepAlivesEnabled(false)` Ôćĺ `srv.Shutdown(10s)` Ôćĺ `wg.Wait()` Ôćĺ `eventBus.Close()` Ôćĺ `database.Close()`. Error from listen goroutine handled via channel, not os.Exit(1). Server timeouts: ReadHeader=10s, Read=30s, Idle=120s (WriteTimeout=0 for SSE).

**Data**
- **Flow:** parse flags (`--config`, `--port`) Ôćĺ config path Ôćĺ load Ôćĺ open Ôćĺ migrate Ôćĺ init queries Ôćĺ LLM client Ôćĺ agent registry Ôćĺ tool registry Ôćĺ history repo Ôćĺ event bus Ôćĺ engine Ôćĺ handler Ôćĺ routes Ôćĺ HTTP goroutine Ôćĺ signal/error Ôćĺ graceful shutdown
- **Config:** `config.ServerConfig{Host, Port, APIKey}`, `config.LLMConfig{Provider, BaseURL, APIKey, Model, MaxIterations}`
- **Middleware chain:** AuthMiddleware (bearer-token, skips /health) Ôćĺ RecoveryMiddleware Ôćĺ CORSMiddleware Ôćĺ LoggingMiddleware
- **DB:** SQLite via `ncruces/go-sqlite3`, pooled connections, WAL mode
- **Handler fields:** `LLM`, `DB`, `SQLDB`, `Config`, `Engine`, `EventBus`, `History`, `Wg`
- **Routes:** health, session CRUD, async chat (202), SSE events (global + session), branches, fork, abort, agents, memory, models, OpenAI compat, message edit ÔÇö 22+ endpoints
- **Shutdown seq:** SetKeepAlivesEnabled(false) Ôćĺ srv.Shutdown(10s) Ôćĺ wg.Wait() Ôćĺ eventBus.Close() Ôćĺ database.Close()
- **Imports:** `sync`, `syscall` added

**Exports**

**Imports**

### `cli/`
CLI entry point for GoPengAI. Uses Cobra framework. 4 subcommands: chat, session, agents, memory.

#### `main.go`
Full-featured CLI client (1028 lines) for the GoPengAI API server. Supports SSE streaming chat, session management with branching, agent listing, and memory access.

**Algorithm**
**Root command**: Setup Cobra, parse `--server` (default `http://localhost:8080`) and `--api-key` flags. PersistentPreRun trims server URL suffix.

**Chat command** (`chat`):
- Sync mode (default): create session if none, POST `/session/{id}/message` (202 accepted).
- SSE streaming mode (`--stream`): subscribe to `GET /session/{id}/events`, send message, parse SSE events ÔÇö `message.part.added` prints tokens, `message.complete` prints model+usage, `message.tool.*` shows tool calls, errors abort.
- REPL mode: interactive loop with `/quit`, `/exit`, `/session`.

**Session command** (`session`):
- `list`: GET `/session`, formatted table (ID, Title, Status, Msgs).
- `show <id>`: GET `/session/<id>`, details + messages.
- `create`: POST `/session` with `--title`/`--agent` flags.
- `delete <id>`: DELETE `/session/<id>`.
- `branches <id>`: GET `/session/{id}/branches`, formatted table (ID, Role, Agent, Content).
- `fork <id>`: POST `/session/{id}/fork` with `--message/-m`, `--content/-c`, `--title/-t`, `--agent/-a` flags.
- `switch <id>`: PUT `/session/{id}/branch` with `--leaf/-l` flag.

**Agents command** (`agents`):
- `list`: GET `/agents`, formatted table (Name, Model, Tools, Description).
- `info <name>`: GET `/agents/{name}`, Name/Model/Parent/Mode/Description/Tools.

**Memory command** (`memory`):
- `list [--agent/-a NAME]`: GET `/memory?agent=NAME`, table (Key, Category, Value).
- `get <key> [--agent/-a NAME]`: GET `/memory/{key}?agent=NAME`, Key/Agent/Category/Value.

**HTTP helpers**: `apiGET`, `apiPOST`, `apiPATCH`, `apiPUT`, `apiDELETE`, `apiGETRaw`. All via `newRequest()` which attaches `Authorization: Bearer <key>` if `--api-key` set.

**SSE parsing**: `parseSSE` reads event blocks. `readSSELine` reads `\n`-delimited lines. SSE raw response uses `text/event-stream`.

**Data**
- **Structures**: `SessionView`, `MessageView`, `AgentView`, `MemoryView`, `SSEEvent`, `MessageCompleted`, `ModelUsage`, `ContentPart` ÔÇö mirror server JSON responses.
- **Types**: Cobra commands, `serverURL`/`apiKey` globals, `httpClient`.
- **Flow**: CLI args Ôćĺ Cobra parse Ôćĺ route to subcommand Ôćĺ HTTP request Ôćĺ JSON/SSE parse Ôćĺ print.

**Exports**
- `main()` entry point
