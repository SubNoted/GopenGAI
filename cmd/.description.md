### `api/`
API server entry point.

#### `main.go`
Entry point ÔÇö loads config, opens DB, runs migrations, init queries, creates LLM client, initializes agent/tool registries, creates history repo + event bus + engine, builds handler, registers routes, logs endpoints, starts HTTP server with full graceful shutdown (no os.Exit).

**Algorithm**
Load config Ôćĺ open DB Ôćĺ migrate Ôćĺ init queries Ôćĺ create LLM client Ôćĺ init agent registry Ôćĺ init tool registry (4 tools) Ôćĺ create history repo Ôćĺ create event bus Ôćĺ create engine (no longer needs SQLDB) Ôćĺ build handler (includes Engine, EventBus, History, Wg) Ôćĺ register routes (22+ endpoints) Ôćĺ start HTTP server in goroutine, send error to buffered `serverErr` chan Ôćĺ select on `quit`+`serverErr` Ôćĺ graceful shutdown: `SetKeepAlivesEnabled(false)` Ôćĺ `srv.Shutdown(10s)` Ôćĺ `wg.Wait()` Ôćĺ `eventBus.Close()` Ôćĺ `database.Close()`. Error from listen goroutine handled via channel, not os.Exit(1). Server timeouts: ReadHeader=10s, Read=30s, Idle=120s (WriteTimeout=0 for SSE).

**Data**
- **Flow:** config path Ôćĺ load Ôćĺ open Ôćĺ migrate Ôćĺ init queries Ôćĺ LLM client Ôćĺ agent registry Ôćĺ tool registry Ôćĺ history repo Ôćĺ event bus Ôćĺ engine Ôćĺ handler Ôćĺ routes Ôćĺ HTTP goroutine Ôćĺ signal/error Ôćĺ graceful shutdown
- **Config:** `config.ServerConfig{Host, Port, APIKey}`, `config.LLMConfig{Provider, BaseURL, APIKey, Model, MaxIterations}`
- **Middleware chain:** AuthMiddleware (bearer-token, skips /health) Ôćĺ RecoveryMiddleware Ôćĺ CORSMiddleware Ôćĺ LoggingMiddleware
- **DB:** SQLite via `ncruces/go-sqlite3`, pooled connections, WAL mode
- **Handler fields:** `LLM`, `DB`, `SQLDB`, `Config`, `Engine`, `EventBus`, `History`, `Wg`
- **Routes:** health, session CRUD, async chat (202), SSE events (global + session), branches, fork, abort, agents, memory, models, OpenAI compat, message edit ÔÇö 22+ endpoints
- **Shutdown seq:** SetKeepAlivesEnabled(false) Ôćĺ srv.Shutdown(10s) Ôćĺ wg.Wait() Ôćĺ eventBus.Close() Ôćĺ database.Close()
- **Imports:** `sync`, `syscall` added

**Exports**
- `main()` ÔÇö starts HTTP server, graceful shutdown on signal

**Imports**
- `gopengai/internal/api` ÔÇö route registration, handler struct, EventBus, NewEventBus
- `gopengai/internal/agent` ÔÇö Engine, Registry, NewEngine, InitializeFromDir
- `gopengai/internal/config` ÔÇö `Load()`, config structs

### `cli/`
CLI entry point for GoPengAI. Uses Cobra framework.

#### `main.go`
Interactive CLI chat client. Sends messages to API server, manages sessions via REST. Supports single message mode and REPL mode.

**Algorithm**
**Root command**: Setup Cobra, parse `--server` flag. PersistentPreRun trims server URL suffix.

**Chat command** (`chat`):
- Single message: create session if none, send message, print response content.
- REPL: loop read stdin lines. Handle `/quit`, `/exit`, `/session`. Send each line, decode response, print content. Create session on first input.

**Session command** (`session`):
- `list`: GET `/session`, decode array, print formatted table (ID, Title, Status, MsgCount).
- `show <id>`: GET `/session/<id>`, decode view, print details + messages (truncated).
- `create`: POST `/session` with title/agent, print ID/title.
- `delete <id>`: DELETE `/session/<id>`, print status.

**HTTP helpers**: GET, POST (marshal body), DELETE wrappers using `serverURL`.

**Utils**: `messageTrunc` rune-aware truncation for titles.

**Data**
- **Structures**: `SessionView` (id, agent_name, title, status, message_count, created_at, updated_at, messages), `MessageView` (id, role, content, agent_name, created_at), `ChatResponse` (session_id, message_id, role, content, model, error).
- **Types**: Cobra commands, `serverURL` global string.
- **Flow**: CLI args Ôćĺ Cobra parse Ôćĺ route to subcommand Ôćĺ HTTP request Ôćĺ JSON decode Ôćĺ print/interactive loop.

**Exports**
- `main()` entry point

**External Dependencies**
- `github.com/spf13/cobra`
