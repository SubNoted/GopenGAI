### `cmd`
Directory with CLI and API entry points.

---

### `api/`
API server entry point.

#### `main.go`
Entry point — loads config, opens DB, runs migrations, init queries, creates LLM client, builds handler, registers routes, logs endpoints, starts HTTP server.

**Algorithm**
Load config from `gopengai.json` (or `os.Args[1]` override). Open SQLite DB at `{DataDir}/gopengai.db`. Run migrations. Init sqlc queries. Create LLM client from config. Build `api.Handler` with LLM client, queries, DB, config. Register routes via `api.RegisterRoutes`. Log endpoint URLs. Start HTTP server at `{Host}:{Port}`.

**Data**
- **Flow:** config path → load config → open DB → migrate → init queries → create LLM client → build handler → register routes → log endpoints → listenAndServe
- **Config:** `config.ServerConfig{Host, Port}`, `config.LLMConfig{Provider, BaseURL, APIKey, Model, MaxIterations}`
- **DB:** SQLite via `ncruces/go-sqlite3`, pooled connections, WAL mode
- **Routes:** `/session` (GET/POST), `/session/{id}` (GET/DELETE), `/session/{id}/message` (POST), `/v1/chat/completions` (POST), `/health` (GET)

**Exports**
- `main()` — starts `GoPengAI listening on <host>:<port>`

**Imports**
- `gopengai/internal/api` — route registration, handler struct
- `gopengai/internal/config` — `Load()`, config structs
- `gopengai/internal/db` — `Open()`, `Migrate()`, `New()`, queries
- `gopengai/internal/llm` — `NewClientFromConfig()`, client struct

### `cli/`
CLI entry point for GoPengAI. Uses Cobra framework.

#### `main.go`
Interactive CLI chat client. Sends messages to API server, manages sessions via REST. Supports single message mode and REPL mode.

**Algorithm**
**Root command**: Setup Cobra, parse `--server` flag. PersistentPreRun trims server URL suffix.

**Chat command** (`chat`):
- Single message: create session if none, send message, print response content.
- REPL: loop read stdin lines. Handle `/quit`, `/exit`, `/session`. Send each line, decode response, print content. Create session on first input.

**Session command** (`session`):
- `list`: GET `/session`, decode array, print formatted table (ID, Title, Status, MsgCount).
- `show <id>`: GET `/session/<id>`, decode view, print details + messages (truncated).
- `create`: POST `/session` with title/agent, print ID/title.
- `delete <id>`: DELETE `/session/<id>`, print status.

**HTTP helpers**: GET, POST (marshal body), DELETE wrappers using `serverURL`.

**Utils**: `messageTrunc` rune-aware truncation for titles.

**Data**
- **Structures**: `SessionView` (id, agent_name, title, status, message_count, created_at, updated_at, messages), `MessageView` (id, role, content, agent_name, created_at), `ChatResponse` (session_id, message_id, role, content, model, error).
- **Types**: Cobra commands, `serverURL` global string.
- **Flow**: CLI args → Cobra parse → route to subcommand → HTTP request → JSON decode → print/interactive loop.

**Exports**
- `main()` entry point

**External Dependencies**
- `github.com/spf13/cobra`
