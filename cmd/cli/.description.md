### `main.go`
Interactive CLI client for the GoPengAI API server. Supports SSE streaming chat, session management, agent listing, and memory access.

#### Algorithm
**Root command**: Setup Cobra, parse `--server` (default `http://localhost:8080`) and `--api-key` flags. PersistentPreRun trims trailing `/` from server URL.

**Chat command** (`chat`):
- **Sync mode** (default): create session if none, POST `/session/{id}/message`, print accepted status (server returns 202 async). Simple REPL loop with `/quit`, `/exit`, `/session` commands.
- **SSE streaming mode** (`--stream` flag): subscribe to `GET /session/{id}/events`, read initial `: connected` comment, send message via `POST /session/{id}/message`, parse SSE events in a loop:
  - `message.part.added` → print token to stdout (streaming display)
  - `message.complete` → print model name + token usage (prompt/completion/total), exit
  - `message.tool.started` / `message.tool.completed` / `message.tool.error` → brief tool call indicators
  - `message.error` → print error, exit
  - `session.status` → exit on "idle"
  - `heartbeat` → ignore

**Session command** (`session`):
- `list`: GET `/session`, decode array, formatted table (ID, Title, Status, Msgs)
- `show <id>`: GET `/session/<id>`, decode view, print details + messages (truncated)
- `create`: POST `/session` with `--title`/`--agent` flags, print ID/title
- `delete <id>`: DELETE `/session/<id>`, print status
- `branches <id>`: GET `/session/{id}/branches`, formatted table (ID, Role, Agent, Content)
- `fork <id>`: POST `/session/{id}/fork` with `--message/-m` (source msg ID), `--content/-c` (new message), `--title/-t`, `--agent/-a` flags
- `switch <id>`: PUT `/session/{id}/branch` with `--leaf/-l` (leaf message ID) flag

**Agents command** (`agents`):
- `list`: GET `/agents`, formatted table (Name, Model, Tools, Description)
- `info <name>`: GET `/agents/{name}`, print Name/Model/Parent/Mode/Description/Tools

**Memory command** (`memory`):
- `list [--agent/-a NAME]`: GET `/memory?agent=NAME`, formatted table (Key, Category, Value)
- `get <key> [--agent/-a NAME]`: GET `/memory/{key}?agent=NAME`, print Key/Agent/Category/Value

**HTTP helpers**: `apiGET`, `apiPOST`, `apiPATCH`, `apiPUT`, `apiDELETE`, `apiGETRaw`. All share `newRequest()` which attaches `Authorization: Bearer <key>` if `--api-key` is set. SSE streaming uses raw HTTP client to read `text/event-stream`.

**SSE parsing**: `parseSSE` reads event blocks (event: + data: lines terminated by empty line). `readSSELine` reads one `\n`-delimited line.

#### Data
- **Structures**: `SessionView`, `MessageView`, `AgentView`, `MemoryView`, `SSEEvent`, `MessageCompleted`, `ModelUsage`, `ContentPart` — all mirror server-side JSON responses.
- **Types**: Cobra commands, `serverURL`/`apiKey` globals.
- **Flow**: CLI args → Cobra parse → route to subcommand → HTTP request → JSON/SSE parse → print.
