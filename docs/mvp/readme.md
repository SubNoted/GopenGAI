# Минимально жизнеспособный продукт

Что будет выполнено в рамках курсового.

## Минимум умений

- Принимать запросы (конфигурационные и рабочие)
- В конфигурационных: настройки для конкретного пользователя (выбор модели, источник, все дела)
- В рабочих запросах: айди пользователя, текст или ещё что-то (запрос к *llm*)
- Возможность добавлять *текстовые* документы к рабочим запросам
- Парсинг этих самых документов

- Возможность настроить в конфиге сервиса доступ к какой-то удаленной базе
- Модуль работы с этой базой соответственно (парсинг файлов для rag)

- Запрос отправляется rag модулю
- должна быть реализована векторная база 
- должна быть база пользователей и их настроек, историй диалога, статистики использования...

- Перенаправление запроса к какой-нибудь нейросети polza ai (обожаю этот сервис)

- Из составленного ответа формируется ответный запрос пользователю + отчет

- Модуль логгирования

## В дальнейшем (мясо на скелет): 

- Цепочка мыслей, самопроверки: агент рекурсивно думает над ответом, повторяет запросы, пока не придет к результату 
- Режим точности (самопроверки + оценка уверенности модели в ответе)

- Поиск в интернете (+ подумать, чтобы в интернет не уходили личные данные)

- Парсинг фото, видео, аудио...
- Модели для генерации фото, видео и аудио могут вызываться как напрямую, так и инструментом с помощью llm.
